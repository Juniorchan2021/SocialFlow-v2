**版本：v2.0（规划中）**
**最后更新：2026-03-12**

**开发工具：Claude Opus 编写和维护**

## 1. 产品定位

### 1.1 一句话定义

SocialFlow 是 MPChat 内部团队的**社媒内容质控与增长中心**——在发布前对文案 + 配图进行全维度检测、评分、改写和优化，确保每条内容合规、算法友好、视觉专业，同时通过 SEO 落地页和可分享检测报告为 MPChat 品牌持续引流。

### 1.2 核心三角

```
          评测（深度化）
           /        \
          /   内容    \
         /   +配图     \
        /   全维度评分   \
       ──────────────────
      检查（智能化）  流量（多层获客）
      AI语义合规      SEO落地页
      算法限流预警     报告裂变分享
      图片风险扫描     爆文模板库
```

### 1.3 与 v1.0 的根本区别

| 维度 | v1.0（当前） | v2.0（目标） |
|------|-------------|-------------|
| 检测对象 | 纯文案 | **文案 + 配图**（完整帖子） |
| 检测深度 | 关键词字符串匹配 | AI 语义理解 + 规则引擎双层 |
| 产品流程 | 粘贴 → 看分 → 结束 | **方向 → 示例 → 优化**（三步闭环） |
| 输出物 | 分数 + 文字建议 | 可分享检测报告 + 改写稿 + 设计参考图 |
| 流量能力 | 零 | SEO 落地页 + 报告分享裂变 |
| 图片能力 | 无 | 合规检测 + 平面设计专家级建议 |

### 1.4 目标用户

**直接用户**：MPChat 内部内容运营团队（发布前质检）
**间接用户**：通过搜索引擎或报告分享进入的外部用户（品牌曝光渠道）

### 2.2 核心交互原则：方向 → 示例 → 优化

每一条检测结果都遵循三步式输出，让用户不需要思考、直接行动：

**第三步：优化（给你清晰的行动清单）**

> ✅ 加入具体数字（3天/5分钟/省了200块）
> ✅ 控制在 15-20 字之间

## 3. 功能模块设计

### 模块 A — 智能文案检测引擎

#### A1. 八维度评分体系

每条内容从 8 个维度打分，汇总为一个雷达图：

| 维度 | 说明 | 评分依据 |
|------|------|---------|
| 合规安全 | 是否触发平台违规 | 关键词 + AI 语义双层检测 |
| 互动潜力 | 预期点赞/评论/收藏 | CTA、提问、情绪词、话题标签 |
| Viral 潜力 | 被转发/分享的概率 | 钩子强度、内容公式匹配、情绪张力 |
| 可读性 | 读起来是否舒服 | 段落结构、句长变化、emoji 分布 |
| 算法友好度 | 平台算法是否会推荐 | 平台特定信号（CES/推荐权重/探索页） |
| 搜索 SEO | 是否容易被站内搜索找到 | 关键词布局、标签策略、标题结构 |
| 视觉匹配 | 配图与文案是否协调 | 图文一致性、风格匹配、视觉吸引力 |
| 发布时机 | 当前是否适合发布 | 平台活跃时段、热点借势窗口 |

#### A2. AI 语义合规检测

不只匹配关键词，还理解上下文含义：

**规则引擎（快速层，毫秒级）**
- 保留现有关键词库，扩展至 500+ 违规词
- 新增变体识别：`微信` → `V信 / VX / 威❤ / wx / 薇芯`
- 四级风险等级不变：严重 / 高 / 中 / 低

**AI 语义层（深度层，1-3 秒）**
- 调用 Claude 分析完整上下文
- 识别隐晦表达：如 "老规矩，评论区见" 实际是引流
- 识别谐音/缩写/emoji 暗语
- 输出 `contextualRisk`（低/中/高）+ 具体原因

#### A3. 平台算法模拟

针对每个平台的推荐算法特征进行模拟评估：

**小红书 — CES 模型模拟**
```
CES = 点赞(1分) + 收藏(1分) + 评论(4分) + 转发(4分) + 关注(8分)
```
- 评估内容引发各项互动的可能性
- 检查笔记 SEO：标题关键词 + 正文关键词密度 + 标签覆盖
- 检查内容结构：首图吸引力描述 + 分段 + emoji + 总结式结尾

**Twitter/X — 推荐算法适配**
- 检查 Thread 结构（长内容拆分质量）
- 互动信号预测：是否能引发 Quote/Reply
- 发布时间建议（基于目标时区活跃高峰）

**Facebook — News Feed 算法适配**
- 有意义互动权重（评论 > 点赞 > 分享）
- 原创内容 vs 转发内容的权重差异
- "See More" 截断点前的钩子质量

**Instagram — 探索页算法适配**
- Caption 长度与互动关系（最佳 125-150 字符）
- Hashtag 策略（大词+中词+小词金字塔结构）
- Reels 文案的前 3 秒钩子评估
- 轮播图每页的文字可读性

**YouTube — 推荐算法适配**
- 标题 CTR 预测（是否使用高点击公式）
- 描述前两行（折叠前可见内容的质量）
- 标签关键词覆盖度
- 缩略图文字与标题的互补性

#### A4. 标题 A/B 评分对比

用户可输入 2-3 个备选标题，系统对每个标题：
- 独立打分（情绪强度 / 好奇心缺口 / 关键词覆盖 / 字数合规）
- 给出推荐排名
- 提供第四个 AI 生成的"最优标题"作为参考

#### A5. 爆款潜力预测

基于以下信号综合预测：
- 内容公式匹配度（是否符合经过验证的爆文结构）
- 热点借势得分（内容是否与当前热点相关）
- 情绪张力分数（内容的情绪起伏强度）
- 平台偏好匹配（内容类型是否是该平台的高权重类型）

输出：爆款概率（低/中/高）+ 提升建议

### 模块 B — 配图智能检测与设计建议

这是 v2.0 最核心的新增模块。用户上传社媒配图（1-9 张），系统从合规和设计两个角度进行全面检测。

#### B1. 配图合规检测

通过 Claude Vision 分析每张图片：

**检测项：**

| 风险类别 | 检测内容 | 示例 |
|---------|---------|------|
| 文字违规 | 图片上的文字是否包含违规信息 | 图上写了微信号/二维码/价格引导 |
| 品牌侵权 | 是否包含未授权的第三方品牌 Logo | 图片出现竞品 Logo、名人肖像 |
| 敏感内容 | 是否有平台不允许的视觉元素 | 过度暴露、暴力暗示、争议符号 |
| 虚假宣传 | 图片是否有夸大/误导性的视觉效果 | P 图过度、虚假对比图、伪造数据截图 |
| 水印残留 | 是否残留其他平台水印 | 抖音水印发到小红书会被限流 |
| 二维码/链接 | 是否包含引流二维码或 URL | 小红书严格禁止图片内嵌二维码 |

**输出格式（方向 → 示例 → 优化）：**

> **方向**：第 2 张图左下角检测到疑似二维码元素，小红书对含二维码图片直接限流
>
> **示例**：[标注问题区域的预览图] — 红框标出问题位置
>
> **优化**：
> ✅ 裁剪或遮盖二维码区域
> ✅ 将引导信息改为"主页有链接"的文字表述
> ✅ 使用小红书官方"商品标签"功能替代

#### B2. 平面设计专家级建议

以顶级营销平面设计师的视角，对每张配图给出**直接可操作、一目了然**的设计建议：

**评估维度：**

| 维度 | 检测内容 | 评分标准 |
|------|---------|---------|
| 视觉层次 | 主体是否突出，视觉动线是否清晰 | 主体占比、留白比例、对比度 |
| 配色方案 | 色彩是否和谐，是否符合平台审美 | 主色调、配色数量、色彩情绪 |
| 文字排版 | 图上文字是否易读、美观 | 字号、行距、对齐、字体选择 |
| 品牌一致性 | 是否符合 MPChat 视觉规范 | Logo 使用、品牌色、风格统一 |
| 平台适配 | 尺寸比例是否最优 | 各平台推荐尺寸、安全区域 |
| 信息密度 | 图片承载的信息量是否合适 | 文字量、元素数量、视觉复杂度 |
| 情绪传达 | 图片传达的情绪是否与文案匹配 | 色温、人物表情、场景氛围 |
| 首图吸引力 | 首图是否能在信息流中脱颖而出 | 对比度、色彩饱和度、视觉冲击力 |

**输出格式（方向 → 示例 → 优化）：**

> **方向**：首图视觉层次不够突出，信息流中容易被忽略。小红书首图的平均停留时间仅 0.3 秒，需要更强的视觉冲击。
>
> **示例**：
> - 参考风格 A：大字标题居中 + 纯色背景 + 产品小图（适合教程类）
> - 参考风格 B：人物特写 + 情绪化表情 + 对比色文字（适合种草类）
> - 参考风格 C：信息图表 + 数据可视化 + 品牌色系（适合评测类）
>
> **优化行动清单**：
> ✅ 将标题字号放大至画面 1/3 高度
> ✅ 使用 MPChat 品牌蓝(#0071E3)作为强调色
> ✅ 主体元素放在画面中心偏上（符合拇指滑动视觉焦点）
> ✅ 添加 3px 白色描边增强文字可读性
> ✅ 首图尺寸调整为 3:4（1080x1440px）适配小红书信息流

#### B3. 图文一致性检测

检测配图与文案之间的匹配度：

- **主题一致**：文案讲支付卡，图片是否展示了卡片/支付场景
- **情绪一致**：文案是轻松种草风，图片是否也是明亮温暖的风格
- **信息互补**：图片是否补充了文案未提及的信息（好），还是重复了文案内容（浪费）

#### B4. 各平台配图尺寸与规范速查

| 平台 | 推荐尺寸 | 图片数量 | 注意事项 |
|------|---------|---------|---------|
| 小红书 | 3:4 (1080x1440) | 1-9张 | 首图决定点击率，避免纯文字图 |
| Twitter | 16:9 (1200x675) | 1-4张 | 单图比多图互动率高 30% |
| Facebook | 1:1 (1200x1200) | 1-10张 | 轮播图用于讲故事 |
| Instagram | 1:1 (1080x1080) 或 4:5 (1080x1350) | 1-10张 | 4:5 在信息流占比最大 |
| YouTube 缩略图 | 16:9 (1280x720) | 1张 | 必须有人脸+大字+高对比色 |

### 模块 C — 一键智能改写

从"检查出问题"到"帮你解决问题"，闭环体验。

## 4. 平台语言策略与检测规则详设

### 4.0 MPChat 各平台语言策略总览

MPChat 在 5 个平台上的语言策略各不相同，检测系统必须据此适配：

| 平台 | 语言 | 账号情况 | 检测语言逻辑 |
|------|------|---------|-------------|
| **小红书** | 纯中文 | 1 个中文账号 | 中文违规词库 + 中文钩子词 + 中文爆文公式 |
| **Twitter/X** | 中文 + 英文 | 2 个账号（中文号 + 英文号） | 自动识别语言 → 切换对应规则集 |
| **Facebook** | 纯英文 | 1 个英文账号 | 英文规则 + crypto行业特殊合规 |
| **Instagram** | 纯英文 | 1 个英文账号 | 英文规则 + 英文Hashtag策略 |
| **YouTube** | 纯英文 | 1 个英文频道 | 英文规则 + 英文SEO + 英文缩略图文字 |

**语言自动识别机制**：
- 系统根据输入内容自动判断中/英文（中文字符占比 > 30% 即判定为中文）
- Twitter 平台额外提供手动切换："我在发中文号" / "我在发英文号"
- 不同语言加载不同的：违规词库、钩子词库、爆文公式、改写Prompt、设计建议

**对 UI 和输出的影响**：
- 小红书：检测结果全部中文输出
- Twitter 中文号：检测结果中文输出，但提醒符合 Twitter 全球政策（英文政策）
- Twitter 英文号 / Facebook / Instagram / YouTube：检测结果英文输出
- 配图设计建议：根据图片上的文字语言给出建议（中文排版规则 vs 英文排版规则）

### 4.1 小红书（纯中文）

**合规规则（500+ 中文违规词）：**
- 严重违规：加密货币（crypto/比特币/以太坊/币圈/交易所）、博彩、传销、政治敏感（直接封号）
- 高风险：导流词（微信/VX/V信/威❤/二维码/加群/进群/私域/telegram/tg）
- 中风险：诱导互动（求赞/互粉/一键三连/双击/关注有礼）
- 低风险：夸大宣传（最便宜/绝对有效/百分百/全网最低/保证效果）

**MPChat 特殊注意**：小红书对 crypto/支付卡/虚拟货币类内容管控极严，MPChat 团队发布时需要：
- 避免直接提及 "加密货币/crypto/区块链" 等词
- 用 "数字支付/跨境支付/全球支付卡" 等合规替代表述
- 配图中不要出现 "USDT/BTC/ETH" 等 crypto 符号
- 系统需内置一套 "crypto → 合规替代词" 映射表

**算法信号检测（中文优化）：**
- 标题：10-20 字 + 含中文情绪词（震惊/救命/绝了/宝藏/亲测）+ 含数字 → 高 CES 初始权重
- 正文：150-500 字 + 分段 3+ + emoji 适量 → 读完率高
- 标签：5-8 个中文标签（1 大流量词 + 3 精准词 + 1 地域词）→ 搜索流量
- 互动引导：中文提问结尾（"姐妹们觉得呢？" / "你们遇到过吗？"）→ 评论量提升
- 首图：3:4 竖图 + 大标题（中文字体建议思源黑体/苹方）+ 高饱和色

**配图检测（小红书特有）：**
- 首图是否有吸引力（是否会被划走）
- 是否包含其他平台水印（抖音/快手水印会限流）
- 图片上的中文文字是否清晰可读（字号、行距、字体选择）
- 图片风格是否符合小红书审美（明亮、干净、生活感）
- 图片上是否有 crypto 相关文字/符号/Logo（小红书高风险）

**中文爆文公式库：**
- [数字] + [动作] + [惊喜结果]："用了30天，皮肤真的白了！"
- [情绪词] + [核心发现]："救命！这个5块钱的东西太好用了"
- [身份认同] + [专属攻略]："打工人早餐攻略，5分钟营养不将就"
- [平替/对比] + [省钱逻辑]："花50替代500效果，省下的钱买奶茶"
- [问题] + [我的解决方案]："跨境付款手续费太贵？这个方法省了80%"

### 4.2 Twitter / X（中文号 + 英文号 双轨检测）

Twitter 是唯一需要支持双语言的平台。系统需提供语言切换或自动识别。

#### 4.2.1 Twitter 中文号规则

**合规规则（中文 + Twitter 全球政策）：**
- 严重违规：仇恨言论、恐怖主义、自残引导（Twitter 全球通用）
- 高风险：买粉/假互动/批量DM/操纵行为
- 中文特有风险：涉政敏感词、中文仇恨言论
- 互动诱饵：关注互关/互赞/转发抽奖（Twitter 限制）

**算法信号检测（中文推文优化）：**
- 字符优化：中文推文 70-140 字最佳（中文 1 字 = 约 2 字节，280 字符限制内约 140 中文字）
- Thread 结构（中文）：首推中文钩子 + 编号 + 尾推引导关注
- 中文 Hashtag：2-3 个（#跨境支付 #数字游民 #全球支付卡）
- 互动引导：中文提问（"你们用什么方式跨境支付？"）

**中文推文爆文公式：**
- 观点对立："大家都说XX好，但我觉得YY才是正解"
- 经验总结："做了3年跨境电商，最大的教训是..."
- 数据冲击："98%的人不知道，跨境转账可以免手续费"
- 行业揭秘："圈内人不会告诉你的跨境支付真相"

#### 4.2.2 Twitter 英文号规则

**合规规则（英文 + Twitter 全球政策）：**
- Policy Violation: hate speech, terrorism, CSAM, self-harm guidance
- High Risk: buy followers, fake engagement, mass DM, manipulation, ban evasion
- Engagement Bait: RT for RT, F4F, like for like, retweet if you agree
- Minor: link in bio, DM me, check bio

**算法信号检测（英文推文优化）：**
- 字符优化：100-250 字符最佳互动区间
- Thread 结构：首推 Hook + 编号（1/N）+ 尾推 CTA（"Follow for more"）
- 英文 Hashtag：1-2 个精准标签（#CryptoPayments #DigitalNomad）
- 互动预测：问句结尾 +3x 回复（"What do you think?" / "Agree or disagree?"）

**英文推文爆文公式：**
- Hot take: "[Controversial but defensible statement]"
- Thread: "Here's what nobody tells you about [topic] 🧵"
- Data shock: "I analyzed [X] and here's what I found:"
- PSA: "[Widely believed thing] is actually wrong. Here's why:"
- Experience: "In [X] years of [doing Y], I've learned:"

**配图检测（Twitter 通用）：**
- 图片清晰度（Twitter 压缩后是否模糊）
- 图片文字语言是否与推文语言一致（中文号配中文图，英文号配英文图）
- 16:9 预览裁剪区域内的核心信息是否完整
- 图片上的文字字号是否在 Timeline 预览中可读

### 4.3 Facebook（纯英文）

**合规规则（英文）：**
- Community Standard Violation: hate speech, discrimination, violence, terrorism, fake news, misinformation, conspiracy theory, deepfake
- Reduced Distribution Risk: clickbait, "you won't believe", "shocking truth", "they don't want you to know", share to win
- Engagement Bait: like this post, share if you agree, tag someone who, type yes if, 1 like =
- Minor: click the link, limited time, act now, hurry, buy now

**MPChat 特殊注意**：Facebook 对 crypto 广告有严格政策（需预先获批），但 organic post 限制较少。仍需避免：
- 投资回报承诺（"guaranteed returns"）
- 未经授权的金融建议表述
- 虚假紧迫感（"last chance to buy crypto"）

**算法信号检测（英文优化）：**
- 有意义互动：评论权重 > 点赞 > 分享
- "See More" 前钩子：前 3 行（约 80 英文字符）决定展开率
- 原创加成：纯文字/原创图 > 转发链接
- 问句效应：含问句帖子评论量 +100%（"Have you ever..." / "What would you do if..."）

**英文爆文公式：**
- Personal story + Universal lesson: "I almost gave up on [X]. One conversation changed everything."
- Surprising question: "Why do we spend 40 hours on our career but none figuring out what we want?"
- Before vs After: "This time last year: [Pain]. Today: [Transformation]. What changed?"
- Myth-busting: "Drinking 8 glasses daily? The research says something different."

**配图检测：**
- 英文文字占比（Facebook 广告规则：图片文字 < 20%）
- 英文排版是否专业（字体选择、行距、对齐方式）
- 1:1 构图是否有效利用空间
- 轮播图叙事连贯性

### 4.4 Instagram（纯英文）

**合规规则（英文）：**
- Community Guidelines Violation: hate, violence, nudity, misinformation
- Shadowban Risk: banned hashtags, excessive identical hashtags, third-party automation
- Engagement Bait: follow for follow, like for like, F4F, L4L
- Content Suppression: watermarks from other platforms (esp. TikTok logo)

**MPChat 特殊注意**：Instagram 对 crypto 内容较为宽容，但注意：
- 避免"get rich quick"相关表述
- 金融类 Hashtag 需谨慎（#investing 安全，#getrichquick 被限流）
- Reels 中不要有 TikTok 水印（Instagram 会降权）

**算法信号检测（英文优化）：**
- Caption 结构：Hook (前 125 字符，"See More" 前) + Body + CTA + 空行 + Hashtag 区
- Hashtag 金字塔（英文）：5 大词 (#crypto #payments #fintech) + 10 中词 (#cryptocard #digitalnomad #remotework) + 15 小词 (#stablecoinpayment #cryptolifestyle)
- Reels 文案：前 3 秒英文文字钩子 + 强情绪词
- 轮播图：Slide 1 = 强钩子, Slide 2-9 = 内容价值, Last Slide = CTA ("Save for later" / "Follow @mpchat")
- 发布时间：美国东部时间 11AM-1PM, 7PM-9PM 最佳

**英文 Caption 爆文公式：**
- Listicle: "5 tools that changed how I [do X] (last one is free)"
- Contrarian: "Stop doing [common advice]. Here's what actually works."
- Story hook: "I tried [X] for 30 days. Here's what happened."
- Value bomb: "Save this for later. You'll thank me."
- Relatable: "POV: You just discovered [surprisingly good thing]"

**配图检测（Instagram 核心）：**
- 整体视觉风格一致性（与账号 Feed 审美是否匹配）
- 色调和谐度（滤镜使用建议）
- 4:5 构图是否利用了最大展示面积
- 英文文字排版质量（字体选择——建议 Montserrat/Poppins/Inter，避免中文字体）
- 轮播图第一张是否有 "scroll-stop" 能力
- 英文文字叠加的对比度和可读性
- 是否有 TikTok / 其他平台水印（会被 Instagram 降权）

### 4.5 YouTube（纯英文）

**合规规则（英文）：**
- Community Guidelines: hate, violence, misleading, harmful content
- Copyright Risk: background music, stock footage, third-party brand assets
- COPPA compliance (if content could appeal to children)
- Title misleading ("clickbait" that doesn't match video content)

**MPChat 特殊注意**：YouTube 对 crypto 内容的政策：
- 标题避免 "guaranteed profit" / "secret method to earn"
- 描述需加 financial disclaimer（"This is not financial advice"）
- 缩略图不要出现暗示暴富的元素（金币堆/美元符号泛滥）

**算法信号检测（英文优化）：**
- 标题 CTR 公式：[Number] + [Power word] + [Curiosity gap] ≤ 60 英文字符
  - 好标题："I Tested 7 Crypto Cards for 90 Days — Only 1 Was Worth It"
  - 差标题："Crypto Card Review 2026"
- 描述 SEO（英文）：前 2 行含核心关键词 + 第 3-5 行时间戳 + 相关链接
- 标签策略：5-15 个英文标签，核心词在前（crypto card, virtual card, international payment）
- 缩略图与标题互补：缩略图传递情绪/视觉，标题传递信息，不重复

**缩略图检测（YouTube 核心）：**
- 是否包含人脸（有人脸的缩略图 CTR +30%）
- 英文文字大小是否在手机端可读（≥ 画面高度 1/4，建议粗体无衬线字体如 Impact/Bebas Neue）
- 色彩对比度是否足够（建议互补色：蓝+橙、紫+黄、红+青）
- 背景是否过于复杂（建议简化或虚化）
- 情绪表达是否强烈（夸张表情 > 微笑 > 无表情）
- 高点击色彩（黄/红/蓝 > 灰/棕/绿）
- 英文文字不超过 5 个单词（缩略图文字密度过高会降低点击率）

## 9. 竞品深度分析与差异化

### 9.1 竞品全景图

市面上的相关产品可分为 4 类，没有任何一家覆盖 SocialFlow v2.0 的完整能力：

#### 第一类：小红书违规词检测（中文市场，直接竞品）

| 竞品 | 核心功能 | 定价 | 优势 | 短板 |
|------|---------|------|------|------|
| **零克查词** | 小红书/抖音/快手/B站违规词 | 免费 | 多平台专业词库 | 需登录、纯文字、无配图、无AI |
| **句易网** | 违禁词检测+替换建议 | 基础免费 | 新手友好、替换建议 | 只做关键词匹配、无评分体系 |
| **Reditor 编辑器** | 违禁词+emoji+AI文案 | 免费 | 一站式、50万用户、免登录 | 只做小红书、无配图分析、无评分 |
| **句无忧** | AI驱动违禁词检测 | 免费 | 识别精度高、词库更新快 | 只做文字、无互动/算法评分 |
| **AutoXHS** | 违禁词+AI润色+AI浓度 | 免费 | 有AI浓度检测（创新点） | 只做小红书单平台 |

**市场空白**：中文工具全部只做"文字违规词检测"，无一家做配图检测、设计建议、多维度评分或多平台支持。

#### 第二类：社媒互动预测（英文市场，功能交叉）

| 竞品 | 核心功能 | 定价 | 优势 | 短板 |
|------|---------|------|------|------|
| **Focia** | AI互动预测（80%+准确率） | $20/月起 | 个性化模型学习、多平台 | 不做合规、不支持中文平台 |
| **Viralyst** | 爆款DNA+心理情绪分析 | 免费+付费 | 视频钩子分析、策略拆解 | 偏IG/TikTok、无合规、无中文 |
| **PostLate** | 互动预测+时机建议 | 免费 | 图片识别、发布时机 | 只做Twitter/Instagram |
| **PostBro** | 竞品帖子URL拆解+创意生成 | 免费 | 粘贴URL即可分析 | 偏事后分析、不做发前检测 |

**市场空白**：英文工具有互动预测但不做合规，没人支持小红书等中文平台，没人做"合规+互动+配图"的完整组合。

### 9.3 从竞品中学到的设计启发

| 竞品 | 值得借鉴的设计 | SocialFlow 如何吸收 |
|------|--------------|-------------------|
| **ThumbsWise** | "3层反馈"：一眼测试→3C分析→行动修复计划 | 我们的"方向→示例→优化"三步式与其理念一致，进一步细化 |

| **AutoXHS** | "AI浓度检测"：检测内容是否太像AI写的 | 考虑纳入AI痕迹检测维度，与软文机器人的"人性化改写"协同 |
| **ThumbnailCreator** | "注意力热力图"：可视化视觉焦点 | 配图设计建议中用文字描述视觉焦点区域（热力图需额外技术投入，列为Phase 4） |
| **Reditor** | "免登录+免费"：50万用户的增长密码 | SocialFlow 基础检测保持免费，无需登录即可使用 |

### 9.4 SocialFlow v2.0 的 5 大核心护城河

1. **文案+配图联合检测（全球独家）**：市面上没有任何工具同时检测文案合规性和配图设计质量，更没有检测图文一致性的产品
2. **5 平台 × 中英双语**：唯一同时覆盖小红书+Instagram+YouTube+Twitter+Facebook 的检测工具，且中英文界面切换
3. **方向→示例→优化 三步闭环**：不只给分数和问题列表，直接给改好的文案和具体的设计行动清单
4. **平台算法深度模拟**：不是泛泛的"好/坏"，而是模拟小红书CES、Instagram探索页、YouTube推荐等真实算法信号
5. **检测报告裂变分享**：每次检测生成可分享的报告图片和链接，工具本身就是增长引擎——这是所有竞品都没做的
